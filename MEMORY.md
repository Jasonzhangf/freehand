# MEMORY

- 2026-06-15: Freehand architecture baseline uses `contracts + blocks + orchestrators` layering, with UI and provider semantics explicitly separated.
- 2026-06-15: Provider strategy is Rust-native first: OpenAI-compatible adapter may use `async-openai` or raw HTTP; Anthropic adapter defaults to raw HTTP/SSE until a stable Rust SDK is justified.
- 2026-06-15: Orchestrator crates are pure orchestration only. New helper or semantic functions must be searched for in existing libraries first and, when needed, added to `freehand-blocks` rather than `freehand-reason` or `freehand-node`.
- 2026-06-15: Freehand runtime home is `~/.freehand`; runtime logs, ledgers, replays, cache, and local state should converge there rather than leaking across ad hoc paths.
- 2026-06-15: Development and debugging must start from the function map owner and debug artifacts. If feature truth changes, update function map, architecture docs, project skill workflow, and durable memory in the same change set.
- 2026-06-15: Master/slave meaning is an input-permission configuration: whichever side is configured as master receives user input and dispatches tasks; slave in paired healthy mode only executes paired input and rejects unrelated direct input. Paired input source may be a user or another master.
- 2026-06-15: Each agent has a startup configuration file. Slave mode is chosen by startup config and currently confirmed to require at least listening interface, allowed pairing source IP, and pairing token.
- 2026-06-15: Feature work and bug fixes must both pass three self-checks before closure: information sufficiency, logic closure, and lifecycle completeness. If not satisfied, continue read-only tracing first and ask the user only when local truth cannot close the gap.
- 2026-06-15: Local multiple agents are managed by `config.toml`; one `config.toml` may define multiple local agents.
- 2026-06-15: `contracts.core` owns only cross-module shared semantic types plus shared IDs and error contracts. It excludes config schema, UI projection, and debug/trace envelope. Its types default to serializable, replayable, and persistable.
- 2026-06-15: `provider.semantic` v1 supports OpenAI-compatible and Anthropic. It requires both single-shot and stream interfaces, unified semantic outputs, debug-only raw event retention, explicit capability declaration, and recovery classification with periodic windows expressed in seconds and defaulting to half-hour, five-hour, or daily-midnight windows when provider hints are absent.
- 2026-06-15: `reason.turn` writes truth per turn, projects conversation view from turns, and is the only writer of session truth. Freehand does not stop merely because provider finish_reason is stop/end_turn; stopping requires valid completion schema evaluation.
- 2026-06-15: `node.master-slave` v1 is local one-master one-slave only, pairs via WebSocket handshake, uses fixed pairing source per slave config, and keeps slave listening after pairing loss for re-pairing. Master must be able to delegate work, query progress, directly talk, and subscribe to slave turn output.
- 2026-06-15: `ui.protocol` v1 supports CLI and WebUI, exposes full command/projection/query/subscribe truth, keeps query and subscribe separate, carries explicit source identity fields, shows only terminal projected text, and renders slave turn as WebUI-only separate sub-stream card.
- 2026-06-15: Config core v1 is fixed so far: only `~/.freehand/config.toml`; multi-agent layout uses `[agents.<name>]`; one process starts one agent by CLI-selected name; config changes are restart-only; `allowed_pair_ip` is optional and `pair_token` is an environment-variable reference.
- 2026-06-15: Freehand test policy is three-layered by default: module white-box, module black-box, and project black-box. Each feature map entry must declare all three required test groups, and compile/regression flow must run them under the workspace test umbrella plus architecture gates.
- 2026-06-15: Freehand splits feature ownership from code-call truth. `docs/architecture/feature-map.md` owns feature owner and validation routing; `docs/function-maps/` owns request/response/error mainlines, shared multi-reference function descriptions, and code-bound call tables. No feature implementation is closed unless the function-map binding stays in sync with code.

- 2026-06-15: `reason.rewrite-policy` baseline is now implemented in `freehand-blocks`. Provider usage enters rewrite policy only through `prompt_tokens_from_usage`, which maps `TokenUsage.input_tokens` to prompt pressure and rejects invalid usage. `freehand-testkit::ReasonRuntimeHarness` now provides project black-box coverage from provider semantic output to turn truth to usage-driven rewrite policy.
